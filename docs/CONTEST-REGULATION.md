# Конкурсное задание: суть и регламент (коротко и по делу)

Ниже — выжимка по компетенции и правилам из присланного документа. Без воды, только то, что реально нужно при разработке.

## 1) Общие требования
- **Длительность:** 18 часов, 3 дня.
- **Формат:** практическое выполнение задания, оценка по критериям.
- **Контроль версий:** обязателен Git, работа только через организованный сервер.
- **Спринты и канбан:** задача проходит стадии: Новая → В процессе → Можно проверять → Завершена.
- **Дизайн:** строго по макету, адаптивная верстка.

## 2) Критерии оценки (суть)
- **Архитектура** — правильная структура и разделение слоев.
- **Верстка** — соответствие макету, адаптивность.
- **Клиент‑сервер** — корректные запросы, обработка ошибок.
- **Хранение** — кэш/локальное сохранение.
- **Аппаратные функции** — камера, микрофон, уведомления, виджет, тема.
- **Тестирование (вариатив)** — TDD сценарий для Onboarding.
- **Подготовка продукта (вариатив)** — презентация + подготовка к публикации.

## 3) Архитектура (модуль А)
**Требование:** разделить слои бизнес‑логики, UI и домена.
- UI не должен ломать другие экраны.
- Проект разложить по папкам архитектуры.
- Комментарии обязательны (что это, автор, дата, назначение).

**Связка слоев:**
UI → ViewModel → Repository → Data (API/DB)

## 4) Верстка (модуль Б)
**Требование:** точное совпадение с макетом и адаптивность.
- Без пустых зон.
- Все элементы в границах.
- Корректные отступы и шрифты.

**Обязательные экраны:**
- SplashScreen
- Onboarding (переходы, последняя кнопка → выбор языка)
- Language select
- Sign Up (2 шага, проверки email/пароля)
- Log In
- Main screen (аватар, лидерборд, плитки упражнений)
- Profile + смена темы
- Profile resize photo
- Упражнение: угадай животное
- Упражнение: выбери правильный вариант
- Игра
- Упражнение: аудирование

## 5) Клиент‑сервер (модуль В)
**Требование:** все данные через API, корректная обработка ошибок.
- При отсутствии интернета — экран ошибки по макету.
- Ошибка сервера — диалог, закрываемый пользователем.
- Показ индикатора загрузки.

**Где сервер обязателен:**
- Sign Up / Log In
- Main screen (профиль, лидерборд, упражнения)
- Profile (данные и фото)
- Все упражнения + отправка результатов
- Игра (включая WebSocket)

## 6) Хранение (модуль Г)
**Требование:** кэш и сохранение состояния.
- Язык — сохраняется между перезапусками.
- Локализация по готовым переводам.
- Sign Up — сохранить ввод после перезапуска.
- Log In — сессия сохраняется.
- Упражнения — хранить серию успешных ответов.
- Политика/условия — открыть PDF.

## 7) Аппаратные функции (модуль Д)
- Камера/галерея для аватара.
- Обрезка фото с квадратной областью.
- Смена темы по системной и вручную.
- Локальное уведомление, если не заходил > 1 минуты.
- TensorFlow Lite (распознавание животных).
- AR‑режим для 3D модели (если есть).
- Виджет с позицией пользователя в рейтинге.
- Микрофон + распознавание речи (аудирование).

## 8) Тестирование (модуль Е, вариатив)
TDD для Onboarding:
- Очередь картинок/текстов.
- Кнопки меняются на последнем шаге.
- Переходы зависят от состояния очереди.
- Три коммита: RED → GREEN → REFACTOR.

## 9) Подготовка продукта (модуль Ж, вариатив)
- Работа в ветке `Path-X`.
- Презентация для разработчиков: архитектура, классы, производительность.
- Подготовка к публикации (магазин, иконка, описания, скриншоты).

## 10) Что важно не забыть
- **Нельзя:** приносить носители, телефоны, часы.
- **Интернет:** строго ограничен временем.
- **Если приложение не компилируется** — работа не оценивается.
